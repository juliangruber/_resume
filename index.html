Hi, my name is Julian Gruber.

I was born in Jan 1991 and currently live in Munich, Germany.

I finished school with a 1.9 / A-.

I got an extra sheet honoring all my extra curricular achievements. I gave courses in HTML, CSS and PHP and lead the team behind the latest version of the school's website.

I studied computer science at Technical University Munich until I quit after 3rd semester. I am mostly self taught.

I worked as a freelancer while in school, creating 2 websites.

I worked as an intern for two startups and am now employed at the 2nd one. There are 100 people working there and I made them use GIT, Graphite and StatsD and write tests.

I have 77 open source projects on github, 62 modules in the node package manager and contribute to many other open source projects. I landed a commit in node core.

I gave a talk at ux munich, a usability conference, about bleeding edge usability.

I have the desire to take on different hats.

I speak and write english, german and french (un peu).

I write tests, JavaScript (browser & node), C, C++, Ruby, C#, Java, HTML5 and CSS3 (and latin).

I store stuff in LevelDB, Redis, MySQL, MongoDB, the cloud and my machine.

I <3

  • clean interfaces in code and design
  • small things that do one thing and do it well
  • simplifying by layering abstractions

multilevel

Access a levelUp-instance from multiple processes. LevelDB's design forbids you to access a database instance from multiple processes. This (wip) exports the levelUp interface 1:1.

var db = multilevel.client('http://localhost:3000')
db.set('foo', 'bar')

Repo: juliangruber/multilevel-http, juliangruber/multilevel

level-schedule

A durable job scheduler based on LevelDB that transparently handles crashes.

Schedule(db)
  .job('print', function (payload) {
    console.log(payload);
  })
  .run('print', { some : 'json' }, Date.now() + 5000);

// => (after 5s) { some : 'json' }

level-store

A streaming storage engine based on LevelDB.

Just pipe a stream (e.g. from a download or a file) into it, read from any stream that is already stored or is written to, and resume where you left if something bad happened.

request('http://google.com').pipe(store.createWriteStream('google'));

Source.

leveled

A node.js binding to LevelDB with focus on performance and a minimal api, allowing you to write databases in JavaScript.
